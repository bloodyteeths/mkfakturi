# Годишна Сметка (Annual Account) - Complete Form Structures

Source: Central Register of North Macedonia (crm.com.mk) / Global Group Consulting manuals (mojkonsultant.mk)
Legal basis: Pravilnik za forma i sodrzhina na godishnata smetka (Sluzben vesnik na RM br. 52/11, 174/11, 9/12, 13/12, 60/14)

The annual account package consists of 4 mandatory forms:
1. **Биланс на состојба** (Balance Sheet / Statement of Financial Position) - Form ID 17
2. **Биланс на успех** (Income Statement / P&L) - Form ID 18
3. **ДЕ - Дополнителни податоци за државна евиденција** (Supplementary Data for State Registry) - Form ID 19
4. **СПД - Структура на приходи по дејности** (Revenue Structure by Activity) - Form ID 32

Each form has two amount columns:
- **Тековна година** (Current year)
- **Претходна година** (Previous year)

## 3. СПД - СТРУКТУРА НА ПРИХОДИ ПО ДЕЈНОСТИ (Revenue Structure by Activity)

Form ID 32. This is a tabular form, NOT a fixed AOP-line form. It is a recapitulation table.

### Structure

The СПД-Рекапитулар has 3 columns per row:

| Column | Name | Description |
|--------|------|-------------|
| 1 | **Класа** (Class) | NKD Rev.2.1 numeric code for the activity class |
| 2 | **Назив** (Name) | Descriptive name of the activity per NKD |
| 3 | **Остварени приходи** (Revenue realized) | Amount of revenue earned from this activity |

### Rules

- Uses **NKD Rev.2.1** (Национална класификација на дејности, Revision 2.1) codes
- The company lists each activity from which it earns revenue, one row per NKD class
- The **Главна приходна шифра** (main revenue code) is the activity with the **largest share** of total revenue
- The Central Register automatically determines/updates the main revenue code based on the SPD data
- The main revenue code **cannot be changed** during the year -- it is set annually based on the submitted annual account
- **Validation**: Total revenue from SPD recapitulation must equal: **AOP 201 + AOP 223 + AOP 244** from the Income Statement (i.e., the sum of operating revenue + financial income + total revenue)
  - More precisely: must equal **AOP 201 + AOP 223 + AOP 227** (total revenue per AOP 243)

### Example SPD entry

| Класа | Назив | Остварени приходи |
|-------|-------|-------------------|
| 47.11 | Трговија на мало во неспецијализирани продавници, претежно со прехранбени производи, пијалаци и тутун | 5,000,000 |
| 68.20 | Изнајмување и управување со сопствен или изнајмен недвижен имот | 500,000 |
| | **ВКУПНО** | **5,500,000** |

## 4. ДЕ - ДОПОЛНИТЕЛНИ ПОДАТОЦИ ЗА ДРЖАВНА ЕВИДЕНЦИЈА (Supplementary Data for State Registry)

Form ID 19. Contains AOP codes 601-687. This form provides detailed breakdowns of fixed assets (acquisition cost, depreciation, net book value) and other state-required statistical data.

### Structure Overview

The DE form has **87 AOP codes** (601 through 687) organized into sections covering:

#### Section I: Нематеријални средства (Intangible Assets)
Breakdown of intangible assets by:
- Набавна вредност (Acquisition/cost value) - beginning and end of year
- Амортизација (Accumulated depreciation) - beginning and end of year
- Сегашна вредност (Net book value / carrying amount) - beginning and end of year

Columns typically:
| Column | Description |
|--------|-------------|
| Набавна вредност - почеток | Acquisition cost - beginning of year |
| Набавна вредност - крај | Acquisition cost - end of year |
| Исправка на вредност (амортизација) - почеток | Accumulated depreciation - beginning |
| Исправка на вредност (амортизација) - крај | Accumulated depreciation - end |
| Сегашна (книговодствена) вредност - почеток | Net book value - beginning |
